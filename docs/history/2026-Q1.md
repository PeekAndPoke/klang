# 2026 Q1 — Language, patterns & the editor (Jan–Mar 2026)

The quarter the project grew its own brain. We stopped leaning on strudel-js and built
our own interpreter, our own pattern engine, and the UI to live-code in.

## KlangScript (new language)

- Built a JavaScript-ish interpreter, `KlangScript` (parser on better-parse).
- StdLib: math / string / list / object functions (Kotlin-style conventions).
- Kotlin interop + native function/object registration; later a KSP processor so
  native functions register with almost no boilerplate.
- Named parameters; comparison expressions; array functions with block bodies.

## Pattern language (strudel → sprudel)

- Ported the Strudel DSL onto the KlangScript runtime; huge function buildout.
- GraalJS **compatibility tests**: compile patterns in JS and diff the event stream
  against ours — caught a lot.
- Euclidean rhythms, control patterns, random/choice, continuous patterns,
  `struct()` / `mask()`, polymeter, `n()` done properly.
- The big **part/whole** refactoring + verification pass (matching JS semantics).
- Fixed-point `Rational` math to fight floating-point drift.
- Renamed **strudel → sprudel** — taking our own direction now.

## Audio effects & synthesis

- FM synthesis; filter envelopes.
- phaser, tremolo, vowel(), warmth(), crush, coarse, accelerate/glissando.
- Filter pipeline unification; limiter; `ShortArray`/perf passes.
- ExciterDsl (composable), wired into KlangScript.

## Editor, blocks & web UI

- Klang Studio live-coding editor (CodeMirror), source-location tracking, highlighting.
- klangblocks visual block editor MVP — drag & drop, realtime highlighting, round-trip.
- Mini-notation editor + parser refactor.
- Web UI: real pages, dark theme, sample library + explorer.
- Browser **spectrometer** and **oscilloscope**; FE/BE time sync + diagnostics.

## Branding & housekeeping

- **Motör** engine branding begins; FreqIgnitor; KlangScript intellisense first steps.
- Performance: Rational impl tuning, removing `Long` everywhere (JS pain), `AstIndex`
  for finding nodes from code positions.
