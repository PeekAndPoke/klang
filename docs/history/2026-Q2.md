# 2026 Q2 — Making it *sound* good (Apr–Jun 2026)

If Q1 was about building the brain, Q2 was about the ears. The dominant theme of the
quarter — by a wide margin — was the **sound of the audio engine**: chasing warmth,
killing the "plastic pipe", and tuning almost everything by ear. Alongside that, the
engine became configurable (the Engine DSL) and a lot faster.

## 🔊 Audio engine — the sound (the big one)

This is where most of the quarter went. Round after round of listening, tweaking,
re-listening.

- **Fighting the "plastic pipe"** — a long warmth hunt:
    - white-noise driven analogue **drift**; an `analog` param in the Osc DSL feeding
      several destinations (pitch, filter cutoff, envelope).
    - LPF / HPF saturation + randomization; **filter envelope drift**.
    - analog drift later re-calibrated — seed at *centre* so note attacks land in tune,
      filter movers toned down.
- **SuperSaw rewrite** — mono, analog flyback shape (dropped PolyBLEP, softens high
  notes), gain-weighted-centroid tuning anchor, amplitude jitter instead of detune
  jitter. Onset-variation bug fixed by exempting the centre voice from gain jitter.
- **Oscillator tuning & performance** — many rounds; `Osc.pluck` with full params;
  `Osc.variants(osc1, osc2)` to combine ignitors.
- **Body resonator** — parallel band-pass "body" filter (floor + peaks), models a
  *passive* body that never adds energy. `vowel()` reworked the same way.
- **Engine tuning rounds** — compressor improvements (killed an unsteady gain jump),
  countless small by-ear tweaks.
- **Ignitor DSL + click hunt** — per-voice exciter chains; distort oversampling; more
  distort variants; chasing voice clicks.
- **Filter review** — OnePole cutoff bias documented (kept raw on purpose), NaN guards,
  dedup. Engine stays intentionally raw — no silent clamping "for safety".
- **Deep correctness + performance review** — a long pass across the whole DSP path;
  JVM backend fixed and running smoothly again.

## ⚙️ Engine DSL — configurable engines

- ExciterDsl grew into the **Engine DSL**: configurable engines straight from
  KlangScript (`Engine` / `Stage` objects), a **data-driven voice pipeline**, and
  Vca / Filter character knobs.
- Registration round-trip + KlangScript `@TypeExtensions`; a KSP processor fix.
- Established the Motör engine naming convention (Cylinder, Injection, Ignitor, …).

## ⚡ Performance

- **CycleTime** — replaced `Rational` with a fixed-point `CycleTime` value class for
  sprudel timing; fixed the JS BigInt hot path.
- **Mutable VoiceData** — `SprudelVoiceData` made mutable + single-owner, grouped into
  ~12 `Svd*` sub-objects → leaf clone ~17× faster, killing per-event copy overhead.
- **Worklet wire format** — KSP-generated trust-codec replaced kotlinx.serialization in
  `WorkletContract` → worklet decode 67µs → ~385ns (~174×). kotlinx dropped from the wire.
- Ported the DSP to `Double` arrays to avoid float→double conversion cost on the JVM.

## 🎛 Sprudel (pattern language)

- TimeShiftPattern vs CycleTime boundary-onset duplication fixed (late/early/off/echo).
- Structural cycle-boundary selection fixed for `seq(N)…slow(N)` / `<…>` alternation.
- sprudel pattern rewrites + doc updates.
- sprudel DSL named params.

## 🧩 KlangScript

- **Named parameters** — a big multi-step effort across KlangScript and the DSLs.

  `tetris-remix` (one defines symbols, the other imports them).
