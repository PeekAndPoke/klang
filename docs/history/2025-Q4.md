# 2025 Q4 — Inception (Dec 2025)

The project started on 2025-12-20. The first quarter was about proving the idea: run
strudel-js on the JVM through a GraalVM bridge, synthesize audio from its events, and
get something to actually make a sound.

## Foundations

- Kotlin Multiplatform project set up; idea to keep the audio backend platform-agnostic.
- GraalVM bridge running `@strudel/core` on the JVM for event (hap) generation.
- Module split begins: audio-frontend / audio-backend, strudel in its own module.
- Browser audio backend via KotlinJS `AudioWorkletProcessor` (fighting compiler bugs,
  parked Wasm for later).

## Audio engine — first synthesis

- Oscillators: sine, square, saw, triangle, supersaw (rough, slow at first).
- Voice generation + timed playback + audio rendering from voices.
- ADSR envelopes, delay line, orbits (separate delay lines per voice group).
- Reverb, stereo output + panning, distortion.
- Vibrato / FM via parameter arrays for frequency modulation.

## Samples & soundfonts

- Sound sample banks (percussive + melodic), download + caching, on-demand loading.
- Pitch-matched sample playback; best-file-for-pitch selection for melodic banks.
- Soundfont support begun; SampleIndex rewrite; ADSR/loop logic for sampled voices.

## Lessons

- `n()` is **not** `note()` — cost several hours, but simplified the JS bridge.
- Sending big sample blocks to the AudioWorklet stutters → custom chunked
  serialization in `WorkletContract` fixed it.
