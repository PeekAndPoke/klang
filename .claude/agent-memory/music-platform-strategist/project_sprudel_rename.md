User wants to rename their strudel implementation to "Sprudel" (German for sparkling, vs strudel = vortex).

**Why:** The implementation is diverging from original strudel. Keeping the same name creates confusion and implied
compatibility promises. A new name gives freedom to optimize for Klang's users rather than maintaining parity.

**How to apply:**

- Sprudel is the pattern language name; Klang is the platform name
- Sprudel should be visible where patterns are written ("Sprudel patterns", "Sprudel mini-notation") but not in non-code
  features
- Must credit strudel lineage explicitly and generously (strudel by Alex McLean, inspired by Tidal Cycles)
- Positioning: "inspired by strudel, shares its DNA" -- not "forked from strudel and changed X"
- Do not over-explain divergence; that frames Sprudel as derivative rather than sibling
- Rename priority: do it after onboarding work, since the name only matters to users already engaged enough to notice
- Credits page is the entire community-relations strategy -- generous, prominent, honest attribution
