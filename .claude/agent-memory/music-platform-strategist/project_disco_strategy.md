# Project Disco — strategy locked 2026-04-15

Full plan saved to `/opt/dev/peekandpoke/klang/.claude/vision/project-disco-strategy.md`.

## Why

Solo founder (3–4 hrs/week), cold-start, not naturally social-media. Need a launch that builds long-term brand equity
for Motör without a "look at my new engine" framing, and that survives a founder who would rather be coding.

## How to apply

Treat this plan as the baseline for any future Disco-related questions (mascot, post copy, drop scheduling, metrics
review, monetization escalation). Don't re-derive these decisions; reference the file.

## Key locked decisions

- **Mascot framing:** "inhabitant of the engine," not "mascot of the engine." Original IP. Personality:
  curious/earnest/small. Cannot speak.
- **Mascot name recommendation:** Pip (fallback Motörchen). Decision deferred to founder + handle availability.
- **Scene style:** low-poly 3D, one modular scene with swap slots (background, outfit, particle, lighting, prop).
- **Per-drop build target:** 12–16 hrs total after scene framework exists. >20 hrs = scope creep.
- **Warm-up:** 12–16 weeks, 6 content types rotating, 3 posts/wk Twitter+Bluesky, 1 YT Short every 2 wks.
- **Platform tiers:** Twitter/X + YouTube Shorts + GitHub primary; Bluesky + IG Reels cross-post only; Reddit + HN
  tactical per drop; TikTok conditional; LinkedIn/Mastodon/Discord out for months 1–6.
- **Release model:** archive all drops forever, featured = most recent.
- **HN move:** save Show HN for drop #3 or later, never drop #1.
- **Monetization escalation:** GitHub Sponsors day 1; Patreon only once GitHub Sponsors hits ~€500/mo; quit-job signal
  at €1,500/mo.
- **Metrics gates:** review at drop #3 (pivot or continue) and drop #5 (explicit review with decision criteria).
- **Anti-burnout floor rule:** if 1yr in, <€100/mo sponsor revenue + no energy → plan has run its course, do not grind.
