Klang is a live music engine. Music is code that runs, not audio that plays back. Never frame as a DAW or suggest
features that imply recording/exporting audio files.

**Why:** Founder explicitly clarified (2026-03-20) that the long-run vision is NOT recording music into mp3s or similar.
Music always plays from code. The idea is to build hooks so outside events change the music, and musical events change
the outside world. Earlier "production platform" framing was closer but still implied DAW-shaped output.

**How to apply:**

- Never suggest bounce-to-file, export, timeline, or arrangement features
- Frame the competitive set as interactive music middleware (Reactional, Wwise/FMOD) + live coding tools, NOT DAWs
- Evaluate features by: does this make the engine more reactive, more embeddable, more transparent, or better sounding?
- The web frontend is a product AND a showcase, not the whole story
- Pedagogic value is emergent from transparency, never a separate feature category
