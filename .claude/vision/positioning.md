# Klang Strategic Positioning

*Written: 2026-03-20*
*Status: Decided*

## Core Identity

Klang is a **music production platform** with pedagogic transparency. It is NOT a music learning
platform that happens to have production features.

The distinction matters:

- A learning platform frames users as students. Identity: "I'm learning."
- A production platform frames users as creators. Identity: "I'm making."

Nobody identifies as "music student" for fun. Lots of people identify as "music maker."

## The Pedagogic Dimension

Learning is not a separate mode or module. It is a **design principle** applied to every feature:
the machinery is always visible.

- When you shape a synth, you see the oscillators, envelope, and filter respond
- When a pattern plays, the code highlights in sync
- When you add reverb, you can see the waveform change
- When a note clashes, the system can show you why (intervals, harmonic context)

Think "view source" for music production. Ableton hides everything behind proprietary metaphors
and requires YouTube tutorials. Klang shows its work.

Learning happens as a **side effect of transparency**, not as a curriculum.

## Competitive Positioning

| Competitor                  | Strength                       | What they lack                                                                                  |
|-----------------------------|--------------------------------|-------------------------------------------------------------------------------------------------|
| Ableton / FL Studio / Logic | Deep production capability     | Completely opaque. Learning curve is legendary. Nothing explains itself.                        |
| Sonic Pi                    | Code-based, educational intent | Stayed in "educational tool" framing, limiting its ceiling. Not taken seriously for production. |
| VCV Rack                    | Visual signal flow, modular    | Intimidating. No pattern language. Niche audience.                                              |
| Strudel / TidalCycles       | Powerful live coding           | Thin production story. No mixing, mastering, arrangement.                                       |

**Klang's gap**: A music production environment where the machinery is visible. Not dumbed down.
Not hidden behind tutorials. Just genuinely transparent.

Closest analogy: "Ableton Live, if it had grown up in the live coding world instead of the
hardware world."

## What Was Ruled Out

- **"Yousician for code"** positioning — too narrow, frames as educational toy
- **Learning platform with live coding as advanced layer** — inverts the actual hierarchy
- **Competing with DAWs on feature parity** — wrong game; compete on transparency and
  approachability instead
- **Separate "learning mode" vs "production mode"** — false dichotomy; transparency IS the
  pedagogy

## What This Means for Features

Theory tools (interval training, scale explorer, chord reference) are **production aids**, not
standalone modules. You encounter them when you need them in context:

- "I want to harmonize this melody -- what intervals work?" (not "Time for daily practice")
- "What scale is this pattern using?" (shown in context, not in a separate trainer)

Every feature should pass the transparency test: "Can the user see what this is actually doing
to the sound?"
