# Klang Platform Roadmap

*Last updated: 2026-03-20*
*Identity: Music production platform with pedagogic transparency*

---

## Phase 1: Complete the Creative Loop (current priority)

The minimum viable product is a complete creative loop: start -> create -> shape -> see -> finish -> share.

### Immediate (highest leverage)

1. **Onboarding as production, not education**
    - User arrives, hears something (or starts from a template)
    - First action is creative modification, not "lesson 1"
    - Templates by genre/mood: "start from a beat, make it yours"

2. **Interactive playback with visible code**
    - Pattern code highlights as it plays (already identified as highest-value bridge)
    - This IS the pedagogic transparency — not a separate feature

3. **Make existing features self-evident**
    - The "explain a bit" feedback means the UI doesn't communicate what's possible
    - Affordances, not documentation

### Near-term (production capability)

4. **Effects with visible signal processing**
    - Reverb, delay, distortion with before/after visualization
    - Oscilloscope work already in progress — extend to show effect chains

5. **Multi-track/orbit mixing**
    - Volume, pan, basic EQ per orbit
    - Visual mixer view

6. **Synthesis parameter controls**
    - Knobs/sliders for synth parameters that are also visible in code
    - Bidirectional: change a knob, see the code change; change the code, see the knob move

### Launch gate

7. **Audio export** — render to WAV/MP3
8. **Save/load projects** — persistence (local-first is fine)
9. **Shareable links** — "listen to what I made"

## Phase 2: Depth and Discovery (post-launch)

- Arrangement view (go from loops to structured tracks)
- In-context theory aids (scale/chord suggestions when composing)
- Sample browser and import
- Recording live audio input
- Sprudel language refinements based on user patterns
- Rename to Sprudel (after users are engaged enough to notice)

## Phase 3: Platform (later)

- Account system and cloud persistence
- Community sharing / remix culture
- Collaboration (live or async)
- Mobile experience
- Plugin/extension system
- Embedding Klang in other tools / websites

## Deferred Indefinitely

- Standalone interval/theory trainers (reframed as in-context aids)
- Video tutorial system (transparency should reduce the need)
- Formal strudel.cc partnership (be a good citizen, don't force it)
- Feature parity with professional DAWs (wrong game)

## Open Strategic Questions

1. **Arrangement**: How to go from "cool loop" to "structured track"? Live coding tools
   punt on this. Klang needs a novel answer. Timeline view? Scene-based like Ableton?
   Code-defined structure?

2. **The code wall**: Production-first means some users may never want to see code.
   Is there a fully visual mode that's still Klang (not just another DAW)?
   Or is code-visibility a core identity requirement?

3. **Monetization**: Not urgent, but production platforms have clearer paths than
   learning platforms (pro features, sound packs, collaboration tools).

4. **Community identity**: "Klang producers" vs "Klang learners" — the production
   framing creates a more aspirational identity. How to cultivate this?
